2. WHAT THE SOFTWARE CURRENTLY DOES (SUMMARY)
------------------------------------------------------------
✔ Downloads daily market data (bhavcopy, index data, symbol data)  
✔ Cleans, formats, updates OHLC data  
✔ Stores everything locally in folders (no database)  
✔ Maintains historical data for hundreds of stocks & indices  
✔ Runs seasonality analytics:
   - monthly returns
   - weekday patterns
   - special day effects
   - election period behavior
✔ Contains event-specific data (special days, elections)
✔ UI built in basic HTML (tabs, simple tables, charts)
✔ Handles baskets, watchlists, index-level analysis.

------------------------------------------------------------
3. WHAT UNDERSTANDING IS REQUIRED BEFORE MIGRATION
------------------------------------------------------------
To migrate to MERN you must understand:

1. DATA LAYER MIGRATION  
   - Current system uses local folders + CSVs  
   - In MERN must shift to MongoDB schemas for OHLC, Symbols, Events  

2. BACKEND MIGRATION (Python → Node.js)  
   - Python logic using pandas will need to be rewritten  
   - OR kept as Python microservices  

3. FRONTEND MIGRATION (HTML → React/Next.js)  
   - Build modern UI components  
   - Replace static HTML with dynamic pages  

4. FILE WORKFLOWS → API WORKFLOWS  
   - Many CSV-based processes must become API endpoints  

5. JOB AUTOMATION  
   - Cron scripts must be implemented for daily data updates.

IN ONE LINE:
You need to understand MongoDB schemas, Express APIs, React UI, and converting file-based Python logic into Node services (or Python microservices).
